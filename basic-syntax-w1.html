Basic Python Syntax

Interpreters and Scripts

Python has 2 ways to code it - an interpreter and a script. Python is a language that runs line by line, i.e., it does not convert the entire program into binary in 1 go, it does it line by line, and executers each line at a time, until it encounters an error. Because of this, it can be used as sort of a command line interface, where you just type in a command, press enter, and it will give you the output. This is the Python interpreter. However, scripts let you type in a series of commends, and execute them together. What we will be using are scripts. This means you must save your file in the ".py" extention.

The following sections will cover some basic Python syntax and understanding some key basic concepts.

Identifiers

An identifier in Python is a piece of text, or a name that is used to identify anything in Python. This is not restricted to only letter, and can include capitlal letters, lowercase letters, underscores, and numbers. It is important to note that python is case sensitive. This means that it treats a capital letter different from the same lowercase letter. For example, "Python" and "python" are 2 different identifiers. Python also has certain rules for different identifiers:

  • All identifiers start with lowercase letters, except class names which start with uppercase
  • If you start an identifier with an underscore, it changes its rule to private
  • If there are 2 underscores behind an identifier its rule is set to strongly private
  • If it has 2 underscores after the identifier, it is a language-defined special name>

Indentation

Python script requires indentation to identify blocks of code. In other languages, they may use braces ({...}) to indicate a block of code. A thing to note is that most programming language have different "blocks" of code, in the form of functions, conditionels, or loops (more into that later). But in order to iunderstand what belongs to what block, Python uses semicolon (:) and an indentation, which is nothing but a tab space. Take a look at the following code:

1. i = 0

2. myStr = "hello world"

3. if i == 0:

4.    f = "in if"

5.    print myStr

6. print f

As you can see from above, the statements that have a tab space in between them (f = "inif" and print myStr) show that they are called within the if conditional. We know it is the if conditionel because after the statement is a semicolon, indicating everything after a single indentation will be called within it. Most IDEs automatically does it for you. JSBin also does it automatically.

These individual statements, which together make up a block are called suites. Complex statements, such as if, def, for, class, etc. require a header line and the suite. The headerline begins with the statement, such as if, and then ends with a semicolon, indicating any indented statements after it make up the suite

Quotations and Blank Spaces

Quotation marks are used to denote strings, which are just pieces of text. Python accepts single quotation marks ('...'), double quotation marks ("..."), and triple quotation marks ("""...""" or '''...'''). These are all accepted as long as the statement starts and terminates with it. Using more quotation marks can be for when the string itself contains quotation marks! For example, if the string is:

The capital of China is "Beijing", and the capital of India is "New Delhi"

Then you would use it in the following style:

1. myStr = 'The capital of China is "Beijing", and the capital of India is "New Delhi"'

2. print myStr

If you want to include a blank line when outputting or storing the string you type "\n". For example:

1. myStr = 'The capital of China is "Beijing"\nThe capital of India is "New Delhi"'

2. print myStr

This will output as:

The capital of China is "Beijing"

The capital of India is "New Delhi"

Comments

In general programming, a comment is a line that is not taken in or "considered" by the computer when it is running the code. This is good for developers and teams working together, as they can type, within the comments, what the piece of code does. It is best practice to put comments into every code you write so you or other people can understand it better. Comments are preceded with a hash sign (#), and everything in that line is considered the comment. However, it is only the contents of that line, not the lines below it.
